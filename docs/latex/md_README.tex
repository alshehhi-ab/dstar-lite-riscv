This repository contains a high-\/performance implementation of the {\bfseries{D$\ast$ Lite}} path-\/planning algorithm, specifically optimised for {\bfseries{RV32\+I/\+IMC}} RISC-\/V soft-\/cores running on {\bfseries{FPGA}} hardware.\hypertarget{md_README_autotoc_md1}{}\doxysection{Overview}\label{md_README_autotoc_md1}
Dynamic path planning is a fundamental yet computationally expensive task in autonomous robotics. This project addresses these bottlenecks through a hardware-\/software co-\/design approach, implementing D$\ast$ Lite in C++ for resource-\/constrained environments. By utilising fixed-\/point arithmetic and custom ISA extensions, this implementation achieves the low-\/latency response required for real-\/time navigation.\hypertarget{md_README_autotoc_md2}{}\doxysection{Repository Structure \& Branches}\label{md_README_autotoc_md2}
This project is divided into branches based on the level of hardware integration\+:


\begin{DoxyItemize}
\item $\ast$$\ast${\ttfamily main}$\ast$$\ast$\+: The core C++ implementation of D$\ast$ Lite using purely fixed-\/point integer arithmetic. This branch is target-\/agnostic for RISC-\/V IMC cores.
\item $\ast$$\ast${\ttfamily feature/hw-\/accel}$\ast$$\ast$\+: Contains the code optimised for {\bfseries{RV32\+IMC extensions}}. This branch includes the specialised assembly/intrinsics required to interface with custom FPGA hardware logic for priority queue acceleration.
\item $\ast$$\ast${\ttfamily research/profiling}$\ast$$\ast$\+: Dedicated to performance analysis, containing scripts and benchmarks used to evaluate the speedup of the hardware-\/accelerated version versus the software-\/only baseline.
\end{DoxyItemize}\hypertarget{md_README_autotoc_md3}{}\doxysection{Theoretical Background}\label{md_README_autotoc_md3}
The {\bfseries{D$\ast$ Lite}} algorithm, developed by {\bfseries{Sven Koenig}} and {\bfseries{Maxim Likhachev}}, is an incremental heuristic search method. Unlike static planners, D$\ast$ Lite efficiently repairs previous search results when obstacles are detected, avoiding the need to plan from scratch.\hypertarget{md_README_autotoc_md4}{}\doxysubsection{Key Features}\label{md_README_autotoc_md4}

\begin{DoxyItemize}
\item {\bfseries{Incremental Replanning}}\+: Handles dynamic world changes efficiently.
\item {\bfseries{Fixed-\/\+Point Arithmetic}}\+: Pure Fixed-\/\+Point implementation of the D$\ast$ Lite alorithm, reducing the need for floating-\/point hardware support.
\item {\bfseries{Configurable Heuristics}}\+: Supports Manhattan, Octile, and Chebyshev distances.
\end{DoxyItemize}\hypertarget{md_README_autotoc_md5}{}\doxysection{Hardware Architecture}\label{md_README_autotoc_md5}
The implementation targets a RISC-\/V environment with the following specs\+:
\begin{DoxyItemize}
\item {\bfseries{Core}}\+: RV32\+IMC (Integer, Multiply, Compressed).
\item {\bfseries{Custom ISA Extensions}}\+: Moves bottleneck operations like priority calculations into dedicated hardware logic (see the {\ttfamily hw-\/accel} branch).
\item {\bfseries{Target Platform}}\+: RV32\+IMC on FPGA
\end{DoxyItemize}\hypertarget{md_README_autotoc_md6}{}\doxysection{Target Platform}\label{md_README_autotoc_md6}
This project uses a custom FPGA-\/based RISC-\/V platform built around the {\bfseries{low\+RISC Ibex}} core configured for an RV32\+IMC-\/style environment.
